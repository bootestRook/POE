# 角色锁定说明文档

> 版本：v1.0  
> 用途：用于后续生成该角色的 Idle / Walk / Attack / Cast / Hit / Death 等序列帧时，锁定角色形象一致性。  
> 锚点图：`character_anchor_right_idle.png`  
> 最高原则：**角色一致性优先级高于动作夸张度。**

---

## 1. 角色定位

该角色为项目中的 **right 朝向母版角色**，用于建立后续所有动作序列帧的外观基准。

| 项 | 锁定要求 |
|---|---|
| 角色类型 | 单个 Q 版 2D / 2.5D 角色 |
| 朝向 | 面向屏幕右侧 |
| 视角 | 2.5D / dimetric / 轻俯视 |
| 姿态母版 | Idle 站姿 |
| 背景 | 透明底优先；不可加入场景、UI、道具背景 |
| 角色数量 | 只能有 1 个角色 |
| 武器状态 | 当前角色无武器；后续生成时不得擅自添加武器 |
| 用途 | 作为后续动作帧、sprite sheet、动画测试场景的角色外观锁定依据 |

---

## 2. 外观锁定要点

### 2.1 头部与脸部

| 部位 | 锁定描述 |
|---|---|
| 脸型 | 简化圆脸，浅肤色，侧脸朝右 |
| 五官 | 极简五官，单只淡蓝色圆形眼睛，小短线嘴部 |
| 表情 | 中性、冷静，不做夸张表情 |
| 头身比 | Q 版比例，头部相对较大，身体短小 |

### 2.2 发型

| 部位 | 锁定描述 |
|---|---|
| 主发色 | 白色 |
| 发型结构 | 大块几何硬边发束 |
| 前发 | 额前有明显棱角状白色刘海，覆盖部分额头 |
| 侧发 | 右侧脸附近有一片垂落的白色发片 |
| 后发 | 后方大体块白色长发，向左后方展开 |
| 发线 | 使用深灰蓝色描线，线条清晰 |
| 动作允许变化 | 只允许随动作产生轻微摆动，不允许改变发型结构 |

### 2.3 服装

| 部位 | 锁定描述 |
|---|---|
| 披肩 / 外套 | 棕色披肩，位于肩背部，有明显三角形外轮廓 |
| 上衣 | 棕色上衣结构，胸前有米色内衬区域 |
| 长袖 / 围巾感结构 | 正面有一条米色长条形布料自然下垂 |
| 腰部 | 棕色腰带 / 腰封结构，横向分层明显 |
| 下摆 | 米色与棕色布片形成垂落层次 |
| 裤子 | 深色长裤，接近黑灰色 |
| 鞋子 | 棕色短靴，靴口较宽，底部块面感明显 |
| 动作允许变化 | 布料可轻微摆动，但不能改变服装结构、颜色或层次关系 |

## 4. 视角与构图锁定

| 项 | 要求 |
|---|---|
| 朝向 | 固定面向屏幕右侧 |
| 镜头 | 单角色全身图 |
| 透视 | 轻俯视 2.5D / dimetric |
| 构图 | 角色居中，占画面主体区域 |
| 裁切 | 不得切掉头发、脚、披肩、手臂 |
| 尺寸 | 512×512 或 1024×1024 均可 |
| 缩放 | 后续可统一缩到 256×256 |
| 背景 | 透明底优先，其次纯白底 |
| 画风 | 简洁卡通、扁平色块、清晰描线、低细节、矢量感 |

---

## 5. 允许变化项

后续制作动作帧时，以下内容可以变化：

1. 四肢姿势。
2. 身体重心。
3. 头部轻微上下浮动。
4. 头发尾部少量摆动。
5. 披肩和布料少量摆动。
6. 眼睛位置随头部小幅变化。
7. 脚部落点随动作变化。

变化幅度必须服务于动作表达，不能导致角色身份变化。

---

## 6. 禁止变化项

后续制作动作帧时，以下内容禁止变化：

1. 禁止改变脸型、头身比、体型。
2. 禁止改变发型结构、发色、发束数量和主要轮廓。
3. 禁止改变服装结构、服装层次、服装颜色。
4. 禁止新增武器、饰品、护甲、背包、帽子、角、尾巴、特效。
5. 禁止加入场景、UI、文字、水印、编号。
6. 禁止改变朝向；该母版必须始终面向屏幕右侧。
7. 禁止改成写实、厚涂、像素、3D、日系赛璐璐等其他画风。
8. 禁止动作过度夸张导致身体比例失真。
9. 禁止让头发或披肩遮挡角色关键轮廓。
10. 禁止生成多人或重复角色。

## 7. 后续动作帧统一生成提示词

以下提示词可作为后续所有动作帧的固定前缀。

```text
请以角色锚点图 character_anchor_right_idle.png 作为唯一角色外观参考，生成同一个角色的单帧2D游戏序列帧。必须严格保持角色形象不变：白色几何长发、淡蓝色单眼、浅肤色圆脸、棕色披肩、棕色上衣、米色内衬与长袖、深色长裤、棕色短靴、Q版比例、简洁扁平卡通风、清晰描线、低细节、2.5D/dimetric轻俯视、面向屏幕右侧。只允许改变动作姿势，不允许改变角色设计、发型、服装、配色、体型、视角和画风。单个角色，全身，居中，透明背景或纯白背景，不要场景，不要UI，不要文字，不要武器，不要特效。
```

## 8. Idle 动作生成约束

Idle 动作用于角色默认站立循环。

| 帧数 | 要求 |
|---|---|
| 建议帧数 | 4 帧 |
| 动作幅度 | 小 |
| 核心变化 | 呼吸、重心轻微上下、手臂极小摆动、头发和披肩轻微回弹 |
| 禁止 | 大幅走动、攻击预备、奔跑姿势、表情变化 |
| 循环要求 | 第 4 帧必须能自然回到第 1 帧 |

### Idle 固定动作提示词

```text
生成 idle 待机动作单帧。角色保持面向屏幕右侧的自然站姿，身体放松，双脚轻微分开，双臂自然下垂。动作变化仅表现轻微呼吸、重心上下浮动、头发尾部和披肩少量摆动。不得加入攻击、奔跑、跳跃、施法或夸张动作。
```

---

## 9. Walk 动作生成约束

Walk 动作用于普通移动循环。

| 帧数 | 要求 |
|---|---|
| 建议帧数 | 4 帧 |
| 动作方向 | 向屏幕右侧行走 |
| 核心变化 | 双腿交替、双臂反向摆动、身体轻微前倾、头发与披肩轻微拖拽 |
| 禁止 | 奔跑、冲刺、攻击、跳跃、滑步 |
| 循环要求 | 第 4 帧必须能自然回到第 1 帧 |

### Walk 固定动作提示词

```text
生成 walk 行走动作单帧。角色保持面向屏幕右侧，以普通步速向右行走。双腿自然交替，双臂与腿部反向摆动，身体轻微前倾，头发尾部和披肩随移动方向产生轻微拖拽。动作要清晰但不能夸张，不得改变角色外观。
```

---

## 10. Attack / Cast 动作预留约束

如果后续需要攻击或施法动作，必须先遵守外观锁定，再扩展动作。

| 项 | 要求 |
|---|---|
| 攻击方向 | 默认朝屏幕右侧 |
| 武器 | 当前角色无武器；除非另行确认，不得添加武器 |
| 特效 | 角色帧中默认不包含特效；特效应单独制作 |
| 身体幅度 | 可比 idle / walk 更大，但不能破坏比例 |
| 服装摆动 | 可增加，但不能改变服装设计 |

### Attack / Cast 固定动作提示词

```text
生成 attack/cast 动作单帧。角色保持面向屏幕右侧，基于现有无武器角色形象做动作表现。只允许通过手臂、身体重心、头部朝向和披肩摆动表现动作，不得新增武器、法阵、光效、场景或额外装饰。
```

## 11. 质量检查清单

每次生成动作帧后，必须检查以下项目：

| 检查项 | 通过标准 |
|---|---|
| 角色身份 | 一眼看出是同一个角色 |
| 发型 | 白色几何长发结构未变 |
| 脸型 | 圆脸、单眼、极简五官未变 |
| 服装 | 棕色披肩、棕色上衣、米色内衬、深色裤子、棕色短靴未变 |
| 朝向 | 始终面向屏幕右侧 |
| 视角 | 保持 2.5D / dimetric / 轻俯视 |
| 比例 | Q版头身比未变 |
| 背景 | 无场景、无UI、无文字 |
| 动作 | 当前帧动作清晰可读 |
| 循环 | 同组动作能自然循环 |
| 轮廓 | 全身完整，没有裁切 |

---

## 12. 失败判定

出现以下任意情况，必须判定为失败并重做：

1. 看起来像另一个角色。
2. 发型结构明显变化。
3. 衣服颜色或结构变化。
4. 多出武器、饰品、护甲、背包或特效。
5. 朝向不再是右侧。
6. 视角变成正侧面、正面、背面或俯视过强。
7. 角色比例明显变化。
8. 画风变化。
9. 出现多人、场景、UI、文字、编号。
10. 关键部位被裁切。
