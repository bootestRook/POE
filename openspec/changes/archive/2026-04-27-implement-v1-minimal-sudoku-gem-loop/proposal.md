## Why

本 change 目标是把《V1正式最小循环开发文档.md》中定义的正式项目首个最小可循环版本，固化为 OpenSpec 提案。V1 不是 Demo，也不是一次临时玩法验证；它必须以正式项目的模块边界、配置拆分、数据结构、校验规则和后续扩展方向为基础，完成“刷宝石 -> 看词缀 -> 调盘面 -> 技能表现变化 -> 再刷宝石”的闭环。

当前项目 OpenSpec 尚无 active changes 和 baseline specs，本 change 将作为 V1 正式最小循环的首个规格入口。

## What Changes

### change 目标

- 定义 V1 正式最小循环的产品与系统边界。
- 明确 V1 的模块边界：Core Foundation、Content Rule Data、Gem Board Runtime、Skill Runtime、Combat Runtime、Loot Runtime、Inventory / Storage、Presentation UX。
- 明确配置必须按 `configs/` 子目录拆分，禁止 `all.xxx.toml` 大杂烩配置。
- 明确宝石基础定义与玩家获得的宝石实例分离。
- 明确 9x9 数独盘、`gem_type` 合法性、同行/同列/同宫/上下左右相邻关系。
- 明确 source / target 效果路由、关系系数、导管放大限制和技能最终 modifier 汇总。
- 明确 8 个主动技能宝石、24 个辅助宝石结构、宝石稀有度、随机词缀、掉落、拾取、库存、上盘/下盘、自动释放技能和中文 UI。
- 明确 V1 必须提供可在浏览器打开、可操作的 WebApp 入口，玩家可以在页面中查看库存、检查宝石、调整 9x9 数独盘、预览技能最终效果、开始最小战斗、查看掉落并拾取入库。

### V1 正式最小循环定位

V1 是正式项目第一个最小可循环版本，可以内容量小、数值简单、表现简化，但不能按 Demo 脚本堆叠处理。V1 必须验证同一批宝石因为随机词缀和数独盘关系不同，能够形成明显不同的战斗表现。

V1 的核心循环为：

```text
战斗刷取宝石
-> 拾取并入库
-> 查看基础类型、标签、稀有度、随机词缀
-> 在 9x9 数独盘上调整主动技能宝石与辅助宝石
-> 检查 gem_type 数独合法性
-> 汇总 source / target / relation / power 效果路由
-> 计算技能最终效果
-> 战斗中自动释放已激活技能
-> 继续刷取更优宝石
```

### V1 核心体验

- 玩家通过战斗获得主动技能宝石和辅助宝石。
- 掉落宝石必须可能具有随机词缀，且稀有度覆盖 `normal`、`magic`、`rare`。
- 玩家能在宝石库存中查看中文名称、标签、词缀、可影响对象和当前实际生效对象。
- 玩家能把宝石放入或移出 9x9 数独盘。
- 同行、同列、同宫、上下左右相邻关系会改变技能最终效果。
- 数独盘不是普通摆放限制器，而是效果路由器。
- 辅助宝石不是普通数值插件，而是通过盘面关系、标签过滤、source / target 路由影响主动技能。
- 战斗中自动释放已激活技能，技能表现应随最终 modifier 变化。
- 所有玩家可见 UI 文本必须使用中文。
- 玩家必须能打开 WebApp 主界面，并在同一浏览器体验中完成库存查看、宝石详情、上盘/下盘、合法性提示、高亮预览、战斗、掉落和拾取闭环。

### V1 不做范围

V1 禁止加入以下系统：

- 装备系统；
- 装备随机词缀；
- 地图词缀；
- 腐化系统；
- 复杂货币系统；
- 法力系统；
- 力量 / 敏捷 / 智力成长系统；
- 完整元素抗性系统；
- 完整护甲 / 闪避 / 能量护盾系统；
- 赛季系统；
- 复杂技能树；
- 多角色职业。

这些系统可以保留未来扩展位，但不得进入 V1 的配置、运行时逻辑、UI 流程或验收标准。

### 验收口径

V1 验收时只看是否形成正式最小循环：

- 能通过配置加载和校验启动 V1 所需规则数据。
- 能生成和保存宝石实例，且实例引用基础宝石定义并保存随机词缀。
- 能掉落、拾取、入库主动技能宝石和辅助宝石。
- 能在 9x9 数独盘上上盘/下盘宝石，并检查 `gem_type_1` 到 `gem_type_9` 在同行、同列、同宫内不重复。
- 能计算同行、同列、同宫、上下左右相邻关系。
- 能执行 source / target 效果路由，且同一来源宝石对同一目标宝石同一 stat 只结算一次。
- 能按固定顺序汇总技能最终 modifier。
- 能在战斗中自动释放已激活技能，并让技能表现随盘面和词缀变化。
- 能用中文 UI 展示宝石详情、盘面合法性、技能最终效果预览、掉落提示和战斗 HUD。
- 能打开浏览器 WebApp，并在 WebApp 中完成“刷宝石 -> 看词缀 -> 调盘面 -> 技能表现变化 -> 再刷宝石”的可操作闭环。
- 不能出现 V1 禁止系统的业务入口或玩家可见内容。

## Capabilities

### New Capabilities

- `v1-minimal-sudoku-gem-loop`: 定义 V1 正式最小循环，包括配置与校验、数独宝石盘、宝石定义与实例、随机词缀、掉落与库存、技能效果计算、最小战斗循环、中文 UI 和浏览器 WebApp 可操作入口。

### Modified Capabilities

- 无。当前项目没有 baseline specs，本 change 只新增 V1 最小循环能力。

## Impact

- OpenSpec 影响范围：`openspec/changes/implement-v1-minimal-sudoku-gem-loop/`。
- 实现影响范围：`configs/`、`src/liufang/`、`tools/`、`tests/`、WebApp 工程入口与前端源码。
- WebApp 入口必须复用当前 Python V1 规则层或其最小 API / 适配层，不在前端重新硬编码一套分叉规则。
- 后续实现应按 `tasks.md` 分阶段推进，并保持每个阶段的禁止越界事项。
