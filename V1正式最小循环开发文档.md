# V1 正式最小循环开发文档：数独宝石流放like

## 0. 文档定位

本文档用于对齐项目 V1 的正式开发边界。

V1 的目标不是 Demo，而是正式项目的第一个最小可循环版本。它可以内容量小、数值简单、表现简化，但系统划分、配置拆分、数据结构、模块边界和后续扩展方向必须按照正式项目处理。

V1 必须完成的核心体验是：

> 玩家通过战斗刷取技能宝石与辅助宝石，宝石带有随机词缀；玩家在九宫数独盘上摆放宝石，通过行、列、宫、相邻关系改变技能表现，形成最小构筑循环。

---

## 1. V1 核心目标

### 1.1 V1 必须验证的体验

V1 只验证一件核心事：

> 同一批宝石，因为随机词缀和数独盘摆放关系不同，能够形成明显不同的战斗表现。

也就是说，V1 不是验证“有很多技能”，也不是验证“完整装备系统”，而是验证：

1. 宝石可以掉落；
2. 宝石带有随机词缀；
3. 玩家可以在数独盘中摆放宝石；
4. 摆放必须满足数独规则；
5. 宝石之间可以通过行、列、宫、相邻关系产生联动；
6. 联动会明确改变技能表现；
7. 玩家有继续刷宝石、换宝石、调盘面的动力。

### 1.2 V1 明确不做的内容

V1 暂不实现以下系统：

- 装备系统；
- 装备随机词缀；
- 地图词缀；
- 腐化系统；
- 复杂货币系统；
- 法力系统；
- 力量/敏捷/智力成长系统；
- 完整元素抗性系统；
- 完整护甲/闪避/能量护盾系统；
- 赛季系统；
- 复杂技能树；
- 多角色职业。

这些系统可以在配置结构中预留扩展点，但不能进入 V1 实装范围。

## 2. V1 最小循环

### 2.1 局外循环

```text
查看已有宝石
↓
在数独盘中摆放技能宝石与辅助宝石
↓
检查数独合法性
↓
计算技能最终效果
↓
进入战斗
```

### 2.2 局内循环

```text
自动释放已激活技能
↓
击杀怪物
↓
掉落宝石
↓
拾取宝石
↓
完成本轮战斗
↓
回到局外调整宝石盘
```

### 2.3 刷宝循环

```text
获得新宝石
↓
查看基础类型、标签、随机词缀
↓
判断是否替换现有宝石
↓
重新摆盘
↓
获得新技能表现
↓
继续刷取更优宝石
```

V1 的长期驱动力来自三件事：

1. 更好的基础技能宝石；
2. 更好的辅助宝石；
3. 更适合当前盘面关系的随机词缀。

## 3. 一级模块划分

V1 按正式项目模块划分，不按临时 Demo 脚本堆叠。

### 3.1 Core Foundation：基础框架层

负责通用底层能力，不包含具体玩法规则。

包含模块：

- 配置加载；
- 配置校验；
- ID 与引用解析；
- 随机数与随机种子；
- 事件总线；
- 时间与计时；
- 基础日志；
- 调试开关；
- 本地存档基础。

禁止事项：

- 不写具体宝石规则；
- 不写具体技能规则；
- 不写具体怪物规则；
- 不写 UI 文案；
- 不直接引用具体内容表。

---

### 3.2 Content Rule Data：内容与规则数据层

负责所有静态配置、规则定义和数据校验。

包含模块：

- 玩家属性定义；
- 伤害类型定义；
- 宝石标签定义；
- 主动技能宝石定义；
- 辅助宝石定义；
- 宝石随机词缀定义；
- 词缀生成规则；
- 数独盘规则；
- 技能模板定义；
- 状态效果定义；
- 掉落池定义；
- 中文本地化文案。

禁止事项：

- 不写战斗运行时逻辑；
- 不直接操作实体；
- 不处理输入；
- 不处理渲染；
- 不承担存档逻辑。

---

### 3.3 Gem Board Runtime：数独宝石盘运行层

负责数独盘的运行时计算。

包含模块：

- 九宫格盘面数据；
- 宝石放置/移除；
- 数独合法性检查；
- 行关系计算；
- 列关系计算；
- 宫关系计算；
- 相邻关系计算；
- 宝石 source/target 关系计算；
- 技能最终 modifier 汇总；
- 盘面变化后的重新计算。

禁止事项：

- 不生成掉落；
- 不决定怪物行为；
- 不直接播放技能表现；
- 不写具体 UI 交互。

---

### 3.4 Skill Runtime：技能运行层

负责技能模板的运行时执行。

包含模块：

- 技能实例化；
- 技能冷却；
- 技能释放；
- 投射物生成；
- 范围伤害；
- 命中判定；
- 状态施加；
- 技能标签匹配；
- modifier 应用。

禁止事项：

- 不计算数独合法性；
- 不生成随机词缀；
- 不决定掉落内容；
- 不读取 UI 输入。

---

### 3.5 Combat Runtime：战斗运行层

负责局内战斗循环。

包含模块：

- 玩家实体；
- 怪物实体；
- 怪物生成；
- 怪物移动；
- 生命/死亡；
- 战斗时间；
- 击杀事件；
- 掉落触发；
- 战斗胜负。

V1 战斗目标只需要支撑刷宝循环，不需要复杂关卡目标。

---

### 3.6 Loot Runtime：掉落运行层

负责掉落生成和拾取。

包含模块：

- 掉落池读取；
- 掉落权重计算；
- 宝石基础类型随机；
- 宝石随机词缀生成；
- 掉落物实例化；
- 拾取；
- 入库。

V1 只掉落宝石，不掉落装备、货币、地图、材料。

---

### 3.7 Inventory / Storage：宝石库存层

负责玩家持有宝石的数据管理。

包含模块：

- 宝石实例保存；
- 宝石基础定义引用；
- 宝石随机词缀保存；
- 宝石锁定；
- 宝石筛选；
- 宝石排序；
- 宝石上盘/下盘状态；
- 本地存档。

V1 不做复杂背包格子形状，只做宝石列表和数独盘占位关系。

---

### 3.8 Presentation UX：表现与交互层

负责玩家能看到和操作的内容。

包含模块：

- 宝石盘 UI；
- 宝石详情 UI；
- 宝石词缀展示；
- 数独合法性提示；
- 技能最终效果预览；
- 战斗 HUD；
- 掉落提示；
- 中文文案展示。

所有玩家可见文本必须使用中文。

## 4. 配置文件拆分方案

V1 配置必须拆分独立文件，不能使用 `all.xxx.toml` 这类大杂烩文件。

推荐结构如下：

```text
configs/
  core/
    id_rules.toml
    random_rules.toml

  player/
    player_base_stats.toml
    player_stat_defs.toml

  combat/
    damage_types.toml
    hit_rules.toml
    status_effects.toml

  gems/
    gem_type_defs.toml
    active_skill_gems.toml
    support_gems.toml
    gem_tag_defs.toml
    gem_instance_schema.toml

  sudoku_board/
    board_layout.toml
    placement_rules.toml
    relation_rules.toml
    effect_routing_rules.toml

  skills/
    skill_templates.toml
    skill_scaling_rules.toml

  affixes/
    affix_defs.toml
    affix_spawn_rules.toml
    affix_groups.toml
    affix_tiers.toml

  loot/
    gem_drop_pools.toml
    drop_weight_rules.toml

  localization/
    zh_cn.toml
```

### 4.1 配置拆分原则

| 原则 | 说明 |
|---|---|
| 单文件单职责 | 一个文件只定义一类东西 |
| 定义和实例分离 | 宝石基础定义与玩家掉落实例分离 |
| 标签和效果分离 | 标签只做分类，效果写在 effect 或 stat 中 |
| 中文文案独立 | 内部 ID 不直接显示给玩家 |
| 随机规则独立 | 掉落、词缀、权重不能散落在宝石定义中 |
| 盘面规则独立 | 行/列/宫关系不能写死在技能代码里 |

## 5. 玩家属性定义

V1 玩家属性分为三类：基础属性、战斗派生属性、保留属性。

### 5.1 V1 实装基础属性

| 属性 ID | 中文名 | 类型 | 说明 |
|---|---|---|---|
| `max_life` | 最大生命 | 数值 | 玩家生存核心属性 |
| `current_life` | 当前生命 | 运行时数值 | 战斗中变化，不作为配置词缀目标 |
| `move_speed` | 移动速度 | 百分比/数值 | 控制角色移动速度 |
| `pickup_radius` | 拾取范围 | 数值 | 控制自动拾取范围 |
| `skill_slots_active` | 激活技能数量 | 整数 | 由盘面中主动技能宝石决定 |

### 5.2 V1 实装战斗派生属性

| 属性 ID | 中文名 | 说明 |
|---|---|---|
| `damage_add_percent` | 伤害提高 | 通用增加伤害 |
| `damage_final_percent` | 最终伤害修正 | 独立乘区，用于高价值词缀和代价词缀 |
| `attack_speed_add_percent` | 攻击速度提高 | 影响攻击类技能 |
| `cast_speed_add_percent` | 施法速度提高 | 影响法术类技能 |
| `skill_speed_final_percent` | 技能速度最终修正 | 统一影响技能动作/发射/频率表现 |
| `cooldown_reduction_percent` | 冷却缩减 | 降低技能冷却 |
| `added_cooldown_ms` | 附加冷却 | 增加固定冷却，用于代价词缀 |
| `area_add_percent` | 范围提高 | 影响范围技能 |
| `projectile_count_add` | 投射物数量增加 | 影响投射物技能 |
| `projectile_speed_add_percent` | 投射物速度提高 | 影响投射物速度 |
| `crit_chance_add_percent` | 暴击率提高 | V1 可实现，也可先作为显示属性 |
| `crit_damage_add_percent` | 暴击伤害提高 | V1 可实现，也可先作为显示属性 |
| `status_chance_add_percent` | 状态施加概率提高 | 用于燃烧、冰缓、流血等简单状态 |

### 5.3 V1 伤害类型属性

| 属性 ID | 中文名 | 说明 |
|---|---|---|
| `physical_damage_add_percent` | 物理伤害提高 | 影响物理标签技能 |
| `fire_damage_add_percent` | 火焰伤害提高 | 影响火焰标签技能 |
| `cold_damage_add_percent` | 冰霜伤害提高 | 影响冰霜标签技能 |
| `lightning_damage_add_percent` | 闪电伤害提高 | 影响闪电标签技能 |

### 5.5 玩家初始属性建议

```toml

pickup_radius = 2.0

base_crit_damage = 1.5
```

V1 玩家数值不追求完整成长，只需要能支撑技能、宝石词缀和战斗测试。

## 6. 技能宝石标签定义

技能宝石指主动技能宝石，即盘面中的技能来源。

### 6.1 技能宝石基础标签

| 标签 ID | 中文名 | 说明 |
|---|---|---|
| `gem` | 宝石 | 所有宝石共有标签 |
| `active_skill_gem` | 主动技能宝石 | 表示该宝石会提供主动技能 |
| `gem_type_1` | 1号宝石 | V1 固定为主动技能宝石类型 |
| `loot_gem` | 可掉落宝石 | 可进入宝石掉落池 |

### 6.2 技能来源标签

| 标签 ID | 中文名 | 说明 |
|---|---|---|
| `attack` | 攻击 | 使用攻击速度、攻击类加成 |
| `spell` | 法术 | 使用施法速度、法术类加成 |
| `melee` | 近战 | 近距离技能 |
| `ranged` | 远程 | 远程技能 |
| `bow` | 弓 | V1 可先作为攻击子标签 |
| `summon` | 召唤 | 召唤物或放置物技能 |

### 6.3 伤害类型标签

| 标签 ID | 中文名 | 说明 |
|---|---|---|
| `physical` | 物理 | 物理伤害 |
| `fire` | 火焰 | 火焰伤害 |
| `cold` | 冰霜 | 冰霜伤害 |
| `lightning` | 闪电 | 闪电伤害 |

V1 不建议加入混沌伤害。混沌可以作为后续高阶伤害类型。

### 6.4 技能行为标签

| 标签 ID | 中文名 | 说明 |
|---|---|---|
| `projectile` | 投射物 | 可受投射物数量、速度影响 |
| `area` | 范围 | 可受范围提高影响 |
| `chain` | 连锁 | 命中后可跳转目标 |
| `pierce` | 穿透 | 可穿过敌人 |
| `orbit` | 环绕 | 围绕玩家或目标运动 |
| `nova` | 环形爆发 | 以自身或目标为中心爆发 |
| `dot` | 持续伤害 | 造成持续伤害 |
| `aura` | 光环 | 持续影响范围内对象 |
| `trap_or_mine` | 放置爆炸物 | V1 可用于真菌爆弹类技能 |

### 6.5 技能宝石标签使用规则

技能宝石的标签必须满足以下原则：

1. 标签只表达分类，不直接写数值；
2. 标签是辅助宝石、词缀、盘面关系匹配的依据；
3. 一个技能可以有多个标签；
4. 玩家可见文案必须用中文解释标签影响；
5. 不允许同时出现互斥标签，例如同一技能同时标为 `attack` 和 `spell`，除非后续专门设计混合技能规则。

## 7. 辅助宝石标签定义

辅助宝石不直接释放技能，而是通过盘面关系修改主动技能或其他宝石效果。

### 7.1 辅助宝石基础标签

| 标签 ID | 中文名 | 说明 |
|---|---|---|
| `gem` | 宝石 | 所有宝石共有标签 |
| `support_gem` | 辅助宝石 | 不直接释放技能 |
| `loot_gem` | 可掉落宝石 | 可进入宝石掉落池 |

### 7.2 辅助宝石类型标签

V1 建议保留 8 类辅助宝石类型，但不是全部都要做满内容。

| 标签 ID | 中文名 | V1 定位 |
|---|---|---|
| `gem_type_2` | 资源类辅助 | V1 暂少量保留，不做法力核心 |
| `gem_type_3` | 通用技能改造 | V1 主力辅助类型 |
| `gem_type_4` | 高风险高收益 | V1 少量加入，用于制造刷宝刺激 |
| `gem_type_5` | 伤害类型强化 | V1 主力辅助类型 |
| `gem_type_6` | 属性成长 | V1 暂不实装或只做占位 |
| `gem_type_7` | 转化/缩放 | V1 暂不实装或只做极少量 |
| `gem_type_8` | 技能等级 | V1 可加入少量，强化标签聚合 |
| `gem_type_9` | 行列宫导管 | V1 必须加入，这是数独盘特色核心 |

### 7.3 辅助宝石功能标签

| 标签 ID | 中文名 | 说明 |
|---|---|---|
| `support_damage` | 伤害辅助 | 修改伤害 |
| `support_speed` | 速度辅助 | 修改攻击、施法、技能速度 |
| `support_cooldown` | 冷却辅助 | 修改冷却 |
| `support_area` | 范围辅助 | 修改范围 |
| `support_projectile` | 投射物辅助 | 修改投射物数量/速度 |
| `support_crit` | 暴击辅助 | 修改暴击率/暴击伤害 |
| `support_status` | 状态辅助 | 修改燃烧、冰缓、流血等 |
| `support_level` | 技能等级辅助 | 修改特定标签技能等级 |
| `support_relation` | 盘面关系辅助 | 修改行、列、宫、相邻关系 |
| `support_conduit` | 导管辅助 | 放大或转发盘面效果 |

### 7.4 辅助宝石适用条件标签

辅助宝石必须显式配置适用条件，不能只靠 stat 字段名隐含。

推荐字段：

```toml
[apply_filter]
tags_any = ["projectile"]
tags_all = []
tags_none = []
```

示例：

```toml
[apply_filter]
tags_any = ["projectile"]
```

表示该辅助宝石只影响带有 `projectile` 标签的主动技能。

## 8. V1 技能宝石数量

V1 建议先实现 **8 个主动技能宝石**。

数量判断依据：

1. 少于 6 个，盘面构筑变化不足；
2. 多于 12 个，会把开发重心拉向技能内容堆叠；

## 10. 数独宝石盘规则

### 10.1 V1 盘面规格

V1 使用标准 9×9 数独盘。

```text
9 行
9 列
9 个 3×3 九宫格
```

### 10.2 宝石数字/颜色身份

每颗宝石必须拥有一个 `gem_type`，范围为 `gem_type_1` 到 `gem_type_9`。

V1 规则：

> 同一行、同一列、同一九宫格内，不能出现相同 `gem_type` 的宝石。

这等价于数独规则中的“同数字不能重复”。

### 10.3 gem_type 系统职责

| gem_type | V1 职责 | 说明 |
|---|---|---|
| `gem_type_1` | 主动技能宝石 | 提供技能 |
| `gem_type_2` | 资源辅助 | V1 少量或占位 |
| `gem_type_3` | 通用改造辅助 | 主力辅助 |
| `gem_type_4` | 高风险辅助 | 高收益高代价 |
| `gem_type_5` | 伤害强化辅助 | 元素/物理强化 |
| `gem_type_6` | 属性成长辅助 | V1 预留 |
| `gem_type_7` | 转化/缩放辅助 | V1 预留或少量 |
| `gem_type_8` | 技能等级辅助 | 标签聚合 |
| `gem_type_9` | 盘面导管辅助 | 行/列/宫传播核心 |

注意：这是 V1 的系统职责分配，不代表后续不能扩展具体内容。

### 10.4 V1 盘面关系

V1 只实现四类关系：

| 关系 | 说明 |
|---|---|
| 同行 | 两颗宝石处于同一横行 |
| 同列 | 两颗宝石处于同一纵列 |
| 同宫 | 两颗宝石处于同一个 3×3 九宫格 |
| 相邻 | 上下左右四方向相邻 |

暂不实现：

- 对角相邻；
- 骑士步；
- 全局连接；
- 指定编号大小连接；
- 奇偶连接；
- 顺子连接。

这些属于后续扩展。

## 11. 宝石随机词缀系统

宝石随机词缀是 V1 必须体验。

### 11.1 V1 宝石实例结构

基础宝石定义和玩家获得的宝石实例必须分离。

```toml
[gem_instance]
instance_id = "gem_inst_000001"
base_gem_id = "active_fire_bolt"
gem_type = "gem_type_1"
rarity = "magic"
level = 1
prefix_affixes = ["affix_damage_percent_t1"]
suffix_affixes = ["affix_row_source_power_t1"]
implicit_affixes = []
locked = false
```

### 11.2 V1 词缀类型

V1 只做三类词缀：

| 词缀类型 | 英文 ID | 说明 |
|---|---|---|
| 前缀 | `prefix` | 主要提供技能数值增强 |
| 后缀 | `suffix` | 主要提供盘面关系或代价修正 |
| 固有词缀 | `implicit` | 宝石基础自带，不随机或少量随机 |

V1 暂不做：

- 腐化词缀；
- 装备词缀；
- 地图词缀；
- 多词缀洗练；
- 复杂词缀合成。

### 11.3 V1 宝石稀有度

| 稀有度 ID | 中文名 | 词缀数量 |
|---|---|---:|
| `normal` | 普通 | 0 |
| `magic` | 魔法 | 1 |
| `rare` | 稀有 | 2 |

V1 不做唯一宝石和传奇宝石。

### 11.4 V1 词缀大类

| 大类 | 示例效果 | V1 是否必须 |
|---|---|---|
| 技能数值词缀 | 伤害、速度、冷却、范围、投射物 | 必须 |
| 标签强化词缀 | 火焰技能、攻击技能、法术技能增强 | 必须 |
| 盘面来源词缀 | 同行/同列/同宫 source power | 必须 |
| 盘面接受词缀 | 同行/同列/同宫 target power | 必须 |
| 高风险代价词缀 | 增伤但加冷却、加投射但降伤害 | 必须少量 |
| 属性成长词缀 | 每 X 属性获得 Y | 暂缓 |
| 伤害转化词缀 | 物理转火、物理获得火焰 | 暂缓 |
| 腐化关系词缀 | source/target 互换 | 暂缓 |

### 11.5 V1 盘面词缀定义

V1 必须支持以下 6 个盘面词缀方向：

| stat ID | 中文名 | 说明 |
|---|---|---|
| `source_power_row` | 同行来源强度 | 宝石向同行传播效果的强度 |
| `source_power_column` | 同列来源强度 | 宝石向同列传播效果的强度 |
| `source_power_box` | 同宫来源强度 | 宝石向同九宫格传播效果的强度 |
| `target_power_row` | 同行接受强度 | 宝石从同行接收效果的强度 |
| `target_power_column` | 同列接受强度 | 宝石从同列接收效果的强度 |
| `target_power_box` | 同宫接受强度 | 宝石从同九宫格接收效果的强度 |

V1 暂不做：

```text
connect_gem_2_row
connect_gem_3_column
connect_gem_4_box
```

原因：指定类型连接会显著增加学习成本，应该在玩家理解基础行/列/宫传播后再加入。

### 11.6 V1 词缀示例

```toml
[affix_damage_percent_t1]
gen = "prefix"
name_key = "affix.damage_percent.t1.name"
stat = "damage_add_percent"
value_range = [8, 15]
spawn_tags = [
  { tag = "active_skill_gem", weight = 1000 },
  { tag = "support_gem", weight = 400 }
]

[affix_projectile_count_t1]
gen = "prefix"
name_key = "affix.projectile_count.t1.name"
stat = "projectile_count_add"
value_range = [1, 1]
apply_filter.tags_any = ["projectile"]
spawn_tags = [
  { tag = "active_skill_gem", weight = 500 },
  { tag = "support_projectile", weight = 1000 }
]

[affix_row_source_power_t1]
gen = "suffix"
name_key = "affix.row_source_power.t1.name"
stat = "source_power_row"
value_range = [10, 20]
spawn_tags = [
  { tag = "active_skill_gem", weight = 600 },
  { tag = "support_relation", weight = 1000 },
  { tag = "support_conduit", weight = 1500 }
]

[affix_box_target_power_t1]
gen = "suffix"
name_key = "affix.box_target_power.t1.name"
stat = "target_power_box"
value_range = [10, 20]
spawn_tags = [
  { tag = "support_gem", weight = 1000 }
]
```

### 11.7 词缀互斥组

V1 需要实现词缀组，防止随机出失控组合。

| 词缀组 ID | 中文名 | 规则 |
|---|---|---|
| `damage_scaling` | 伤害缩放 | 同一宝石最多 1 条 |
| `speed_scaling` | 速度缩放 | 同一宝石最多 1 条 |
| `projectile_scaling` | 投射物缩放 | 同一宝石最多 1 条 |
| `relation_source` | 来源传播 | 同一宝石最多 1 条 |
| `relation_target` | 接受传播 | 同一宝石最多 1 条 |
| `risk_reward` | 高风险高收益 | 同一宝石最多 1 条 |

### 11.8 V1 随机词缀生成流程

```text
确定掉落宝石基础类型
↓
确定宝石稀有度
↓
根据稀有度确定词缀数量
↓
根据宝石标签筛选可用词缀
↓
根据词缀权重随机候选
↓
检查词缀互斥组
↓
生成词缀数值范围
↓
保存为宝石实例
```

## 12. 数独盘效果路由规则

### 12.1 核心模型

V1 使用简化版 source / target 模型。

```text
最终传播效果 = 来源宝石效果 × 来源强度 × 目标接受强度 × 关系系数
```

V1 不做复杂表达式解释器，只做固定枚举计算。

### 12.2 Source 与 Target 定义

| 概念 | 说明 |
|---|---|
| Source | 提供效果的宝石 |
| Target | 接收效果的宝石 |
| Relation | 两颗宝石之间的盘面关系，例如同行、同列、同宫 |
| Power | 对应关系下的传播/接受强度 |

### 12.3 V1 默认关系系数

| 关系 | 系数 |
|---|---:|
| 相邻 | 1.25 |
| 同行 | 1.00 |
| 同列 | 1.00 |
| 同宫 | 1.00 |

相邻可以作为行/列关系的加强条件，但 V1 需要避免重复计算失控。

推荐规则：

> 如果两颗宝石同时满足相邻和同行/同列，只按相邻关系计算一次，不额外叠加同行/同列。

### 12.4 V1 效果传播限制

V1 为了降低复杂度，采用以下限制：

1. 辅助宝石可以影响主动技能宝石；
2. 导管宝石可以放大行/列/宫传播；
3. 主动技能宝石不反向影响辅助宝石，除非拥有明确词缀；
4. 同一来源宝石对同一目标宝石同一 stat 只结算一次；
5. 不做无限递归传播；
6. 导管只放大，不产生新的二次传播链。

## 13. 技能最终效果计算顺序

V1 推荐计算顺序：

```text
读取主动技能基础定义
↓
应用主动技能宝石自身随机词缀
↓
查找盘面中可影响它的辅助宝石
↓
应用辅助宝石基础效果
↓
应用辅助宝石随机词缀
↓
应用行/列/宫导管放大
↓
汇总 additive modifier
↓
汇总 final modifier
↓
输出最终技能实例
```

### 13.1 乘区规则

V1 只保留三类数值层：

| 层级 | 示例 | 说明 |
|---|---|---|
| base | 基础伤害、基础冷却 | 技能模板提供 |
| additive | 伤害提高、范围提高 | 同类相加 |
| final | 最终伤害、最终技能速度 | 独立乘算 |

暂不做更多复杂乘区。

## 16. 数据校验规则

### 16.1 宝石校验

| 校验项 | 规则 |
|---|---|

### 16.2 词缀校验

| 校验项 | 规则 |
|---|---|
| 词缀 ID 唯一 | 不得重复 |
| stat 合法 | stat 必须存在于属性定义表 |
| spawn_tags 合法 | 所有生成标签必须存在 |
| value_range 合法 | 最小值不得大于最大值 |
| group 合法 | 所属词缀组必须存在 |
| apply_filter 合法 | 引用标签必须存在 |

### 16.3 数独盘校验

| 校验项 | 规则 |
|---|---|
| 同行不重复 | 同行不能有相同 gem_type |
| 同列不重复 | 同列不能有相同 gem_type |
| 同宫不重复 | 同宫不能有相同 gem_type |
| 主动技能数量合法 | 至少 1 个主动技能宝石 |
| 空盘不可进入战斗 | 没有主动技能时禁止开始 |

## 18. V1 关键设计结论

### 18.1 玩家属性

V1 玩家属性只保留能支撑战斗、技能计算和宝石词缀的最小集合：

```text
最大生命
移动速度
拾取范围
通用伤害
技能速度
攻击速度
施法速度
冷却
范围
投射物
暴击
伤害类型加成
状态概率
```

力量、敏捷、智力、法力、抗性、防御体系暂缓。

### 18.2 技能宝石标签

V1 技能宝石标签分为：

```text
基础标签
来源标签
伤害类型标签
技能行为标签
```

标签必须成为辅助宝石、随机词缀和数独盘关系的共同匹配依据。

### 18.3 辅助宝石标签

V1 辅助宝石标签分为：

```text
基础标签
类型标签
功能标签
适用条件标签
```

辅助宝石不能只写数值，必须声明自己影响什么对象。

### 18.4 技能宝石数量

V1 先实现 **8 个主动技能宝石**。

这个数量足够覆盖主要技能模板，同时不会把项目拖成技能内容堆叠。

### 18.5 辅助宝石数量

V1 先实现 **24 个辅助宝石**。

其中必须包含 3 个行列宫导管宝石。

### 18.6 随机词缀

宝石随机词缀是 V1 必须体验，不是后续内容。

V1 随机词缀至少支持：

```text
技能数值词缀
标签强化词缀
盘面来源词缀
盘面接受词缀
高风险高收益词缀
```

### 18.7 最核心的项目差异点

V1 必须避免做成：

```text
流放辅助宝石 + 数独摆放限制
```

而要做成：

```text
宝石随机词缀 + 数独盘面关系 + 技能表现变化
```

数独盘不是限制器，而是效果路由器。

## 19. 后续可扩展方向

V1 完成后，可以按以下顺序扩展：

1. 更多主动技能；
2. 更多辅助宝石；
3. 指定类型连接：`connect_gem_x_row/column/box`；
4. 腐化词缀；
5. 属性成长；
6. 法力与资源系统；
7. 装备系统；
8. 地图与区域词缀；
9. 赛季机制；
10. 更复杂的数独图案奖励，例如顺子、奇偶、宫内完整 1~9。

扩展时必须保持一个原则：

> 所有新增内容都应优先强化“宝石盘关系”和“刷宝石追求”，而不是把系统拉回普通 ARPG 装备堆数值。
